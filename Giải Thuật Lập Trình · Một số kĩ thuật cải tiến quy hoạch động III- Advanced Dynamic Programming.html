Một số kĩ thuật cải tiến quy hoạch động III- Advanced Dynamic Programming

June 24, 2015 in Uncategorized | No comments

Trong phần này mình giới thiệu kĩ thuật bao lồi (convex hull trick) để cải tiến thuật toán quy hoạch động với thời gian O(n2)O(n2) xuống còn O(nlogn)O(nlog⁡n), thậm chí với một số trường hợp xuống còn O(n)O(n), cho một lớp các bài toán. Trước hết chúng ta hãy xem xét bài toán quy hoạch động tổng quát sau:

Problem 1: Cho một bài toán với nn phần tử khác nhau 1,2,…,n1,2,…,n. Phần tử thứ ii có hai tính chất A[i],B[i]A[i],B[i]. Hàm mục tiêu của bài toán này có thể được biểu diễn thông qua bảng quy hoạch động một chiều C[1,2,…,n]C[1,2,…,n] thỏa mãn hệ thức sau:

C[i]={B[i],minj<i{C[j]+A[i]B[j]},if i=1otherwise (1)C[i]={B[i],if i=1minj<i{C[j]+A[i]B[j]},otherwise (1)

Tìm giá trị C[n]C[n].

 
Từ công thức (1)(1), ta có thể dễ dàng thiết kế giải thuật O(n2)O(n2) để giải bài toán như sau:

SlowDynamic(A[1,2,…,n],B[1,2,…,n]A[1,2,…,n],B[1,2,…,n]):
    C[1]←B[1]C[1]←B[1]
    for i←2i←2 to nn
        tmp ←+∞←+∞
        for j←1j←1 to i−1i−1
            tmp ←min{←min{tmp, C[j]+A[i]B[j]}C[j]+A[i]B[j]}
        C[i]←C[i]← tmp
    return C[n]C[n]

 
Sau đây mình sẽ giới thiệu kĩ thuật bao lồi để giảm thời gian tính của thuật toán xuống còn O(nlogn)O(nlog⁡n).

Kĩ thuật bao lồi
Trước hết chúng ta hãy tạm quên bài toán 1 để xét bài toán hình học sau:

Problem 2: Cho một tập nn đường thẳng D={d1,d2,…,dn}D={d1,d2,…,dn} trong đó (di)yi=aix+bi,ai≥0(di)yi=aix+bi,ai≥0 và kk điểm trên trục OxOx: p1,p2,…,pkp1,p2,…,pk. Tìm q1,q2,…,qkq1,q2,…,qk trong đó mỗi giá trị qℓqℓ được định nghĩa như sau:

qℓ=min1≤i≤nai⋅pℓ+biqℓ=min1≤i≤nai⋅pℓ+bi

 

Với mỗi điểm pℓpℓ, bằng cách duyệt qua tất cả các đường thẳng trong DD, ta có thể tính được qℓqℓ trong thời gian O(n)O(n). Do đó tổng thời gian để tìm tất cả các điểm q1,q2…,qkq1,q2…,qk là O(kn)O(kn). Tuy nhiên, ta có thể tìm các điểm này nhanh hơn:

Lemma 1: Tồn tại một thuật toán tìm tất cả các điểm q1,q2…,qkq1,q2…,qk trong thời gian O((k+n)logn)O((k+n)log⁡n).

 
Định lí 1 đúng cả với trường hợp tập các đường thẳng DD có đường thẳng với ai<0ai<0. Tuy nhiên, trong các ứng dụng với bài toán quy hoạch động, các aiai luôn không âm. Ngoài ra, điều kiện ai≥0ai≥0 cũng làm cho việc phân tích thuật toán trở nên đơn giản hơn. Trước hết chúng ta hãy xem ví dụ với D={−x+y=3,−x+2y=4,D={−x+y=3,−x+2y=4, −x+6y=16,−4x+5y=20}−x+6y=16,−4x+5y=20}. Ta có thể thấy với mỗi điểm (p,0)∈Ox(p,0)∈Ox, giá trị nhỏ nhất tương ứng (0,q)(0,q) thỏa mãn (p,q)(p,q) thuộc phần gạch đỏ. Phần gạch đỏ đó gọi là bao lồi (do đó kĩ thuật này còn gọi là kĩ thuật bao lồi). Như vậy, dựa vào hình vẽ trên, đường thẳng −4x+5y=20−4x+5y=20 trong DD là dư thừa (tung độ của điểm có hoành độ pp trên đường thằng này luôn lớn hơn tung độ của điểm có cùng hoành độ trên một đường thẳng khác của DD).
Dựa vào ví dụ trên, ta thấy nếu chúng ta có một cách biểu diễn bao lồi phù hợp, chúng ta có thể nhanh chóng xác định được tung độ qq nhỏ nhất ứng với hoành độ pp. Ta có thể biểu diễn bao lồi bằng tập các interval và một đường thẳng tương ứng mỗi interval. typedef struct{
    double a;
    double b;
  } double_pair;
 
 struct Stack_node {
    int lineId;
    struct Stack_node *next;
  };
typedef struct Stack_node stack_node;
 
stack_node *S;               // the stack
double_pair D[MAX_SIZE];     // the set of lines
double_pair I[MAX_SIZE];     // the set of intervals
double      Q[MAX_SIZE];     // the set of query points
int         ALines[MAX_SIZE];// the set of lines associated with intervals
 
int find_hull(int n){
    qsort(D, n, sizeof(*D), slope_compare);
    S = malloc( sizeof(stack_node));
    S->lineId = -1; // head of the stack
    push(0);        // push d_1 to the stack;
    I[0].a = -(double)INFTY;
    I[0].b = (double)INFTY;
    int m = 0;
    ALines[m] = 0;
    int i = 0;
    stack_node *d;
    double x = 0.0;
    for(i = 1; i <n ;i++){
        d = peek();
        x = find_intersection_x(d->lineId, i);
        while(x <= I[m].a){  // found a redundant line
            pop();       // remove the redundant line
            m--;
            d = peek();
            x = find_intersection_x(d->lineId, i);
        }
        push(i);             // push d_i to the stack
        I[m].b = x;
        I[m+1].a = x;
        I[m+1].b = (double)INFTY;
        ALines[m+1] = i;
        m++;
    }
 
    return m+1;        // the number of intervals
}
 
double find_intersection_x(int x, int y){
    double_pair d_a = D[x];
    double_pair d_b = D[y];
    return (d_b.b - d_a.b)/(d_a.a - d_b.a);
}

void query(double points[], int k, int n){
    int m = find_hull(n);
    int i = 0, q = 0;
    for(i = 0; i < k; i++){
        q = interval_search(0, m-1, points[i]);
        Q[i] = D[ALines[q]].a*points[i] + D[ALines[q]].b;
    }
}
 
 
int interval_search(int x, int y, double p){
    if(y <= x){
        return x;
    } else {
        int mid = (x+y)/2;
        if((I[mid].a <= p) && (p <= I[mid].b)){
            return mid;
        } else if (p > I[mid].b){
            return interval_search(mid+1, y, p);
        } else {
            return interval_search(x, mid-1, p);
        }
    }
}

        if ℓ<mℓ<m         [[the line associated with IℓIℓ is not redundant]]
            associate IℓIℓ with A[i]A[i]
        else
            if A[i]∈ImA[i]∈Im
                associate ImIm with A[i]A[i]
            else
                associate Im+1Im+1 with A[i]A[i]
        associate Im+1Im+1 with didi
        m←m+1m←m+1
    return C[n]C[n]

 
Phân tích thời gian Ta chỉ cần phân tích thời gian của thủ tục tìm kiếm interval IℓIℓ sao cho A[i]∈IℓA[i]∈Iℓ trong mỗi vòng lắp. Trong trường hợp tồi nhất, chúng ta phải duyệt qua toàn bộ tập các interval. Tuy nhiên, nếu chúng ta đã duyệt qua KK intervals, thì các vòng lặp sau, chúng ta không cần phải xét các interval này và các đường thằng tương ứng với chúng nữa. Do đó, tổng thời gian tìm kiếm interval của thuật toán chỉ là O(n)O(n) và đó cũng là thời gian của cả thuật toán.

Trường hợp tổng quát

Trong trường hợp tổng quát, chúng ta có thể áp dụng thủ tục IntervalBinSearch({I1,I2,…,Im},A[i]{I1,I2,…,Im},A[i]) để tìm kiếm interval chứa A[i]A[i]. Do đó chúng ta mất O(logn)O(log⁡n) cho việc tìm kiếm interval cho mỗi vòng lặp của thuật toán. Vấn đề còn lại chỉ là làm sao để cập nhật bao lồi trong mỗi bước khi ta thêm đường thằng (d)B[i]x+C[i](d)B[i]x+C[i]. Ở đây mình chỉ đưa ra ý tưởng để thực hiện cập nhật bao lồi trong thời gian trung bình O(logn)O(log⁡n) mỗi bước. Do đó, tổng thời gian của thuật toán là O(nlogn)O(nlog⁡n).

Ý tưởng của thuật toán dựa trên thuật toán Sweep Line (mình sẽ giới thiệu trong bài viết sau). Ta sẽ luôn luôn lưu các đường thẳng trong bao lồi theo thứ tự tăng dần của slope sau mỗi bước. Gọi d1,d2,…,dmd1,d2,…,dm là cá đường thẳng trong bao lồi sau bước i−1i−1. Để đơn giản ta giả sử không có hai đường thẳng nào song song. Trong trường hợp có hai đường song song, ta chỉ cần thay đổi thuậtt toán một chút. Bây giờ ta có thêm đường thằng (di)y=aix+bi(di)y=aix+bi, để cập nhật bao lồi trước hết ta thực hiện tìm kiếm nhị phân để tìm ra đường thẳng (dj)y=ajy+bj(dj)y=ajy+bj sao cho aj>ai>aj+1aj>ai>aj+1. Trường hợp j=mj=m, ta cập nhật như trong trường hợp đặc biệt 1. Do đó, ta có thể giả sử j<mj<m. Gọi (xp,yp)(xp,yp) là giao điểm của djdj và dj+1dj+1. Xét đường thằng (d)y=aix+b′i(d)y=aix+bi′ có cùng slope aiai với didi và đi qua (xp,yp)(xp,yp). Nếu b′i≤bibi′≤bi (đường thằng didi có màu xanh trong hình dưới đây), đường thằng didi là dư thừa và do đó, bao lồi không thay đổi khi ta thêm đường thằng didi. Nếu b′i>bibi′>bi, ta lần lượt "dịch" đường thằng dd về phía didi cho đến khi dd trùng với didi. Nếu có sự "thay đổi" của bao lồi (các đường thẳng màu nâu), ta cập nhật lại bao lồi. Ta có thể nhận thấy trong trường hợp tồi nhất, ta phải dịch đường thằng dd O(m)O(m) lần. Tuy nhiên, nếu ta dịch KK lần, thì mỗi lần dịch ta sẽ "loại bớt" được KK đường thẳng dư thừa và các đường thẳng này sẽ không được xét đến trong các vòng lặp tiếp theo. Do đó, thời gian trung bình mỗi bước lặp vẫn là O(logn)O(log⁡n)

Bài toán sau được lấy từ codeforces. Bạn đọc có thể áp dụng phương pháp mình giới thiệu ở đây để nộp bài:
